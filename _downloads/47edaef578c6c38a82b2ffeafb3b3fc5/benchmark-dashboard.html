Methodology. The 36 model-driving e2e tests — the full tests/e2e/ suite minus the 13 files that don't call an LLM and/or don't route through the model under test (hover for the reason: test_llm_providers.py, test_ariel_search.py, test_dockerfile_e2e.py, test_claude_code_build_integration.py, test_llm_channel_namer.py, test_channel_finder_mcp_benchmarks.py, test_mcp_readiness.py, test_sdk_helpers.py, test_dispatch_deploy.py, test_proxy_full_chain_e2e.py, test_proxy_live_roundtrip_e2e.py, test_proxy_open_model_harness_e2e.py, test_dispatch_tutorial.py) — are forced onto each CBORG model suite-wide via OSPREY_E2E_FORCE_PROVIDER=cborg + OSPREY_E2E_FORCE_MODEL (all tiers collapse to one model). Open models route through OSPREY's Anthropic↔OpenAI translation proxy. The per-test timeout is 1800s; a timeout counts as a failure (the model did not complete the task within the cap) and is shown as a distinct category so it can be told apart from a logic failure. Models were chosen from a 14-model lightweight probe to span weak/fast→big/slow across distinct open families (gpt-oss, gemma, cborg). qwen-3 / qwen-3-coder are open-weight but CBORG-proxied (google/ route, not lbl/ self-hosted), included as an additional open family. Dropped at probe: cborg-instant, cborg-instant-short (could not relay a tool result; short context truncates the harness prompt). Closed-weight commercial models (GPT/Gemini/…) are out of scope as study subjects, but single-seed Anthropic Claude reference models — claude-haiku-4-5 (the suite's literal default tier), claude-sonnet-4-6, and claude-opus-4-6 — bracket the open models weak→strong as a control/ceiling, showing how they compare against the models the tests were written for. Open subjects run on the Mac Studio via CBORG; the Anthropic reference rows route natively via als-apg (see the [provider] line under each model).
